# Epic 11 Context

## Title
star-wars 3D-vector render pipeline (clipping + camera/MVP + debug overlay)

## Overview
Fix the star-wars 3D vector renderer so geometry positions, sizes, and reads correctly — WITHOUT a 3D engine. Root causes (see star-wars/docs/adr/0001-3d-vector-render-pipeline.md): (1) no near-plane line clipping in wireframe.ts drawWireframe (edges straddling the camera plane are dropped, so the Death Star surface and trench render as triangles); (2) no camera/view transform or scale — placement is faked with ad-hoc world constants, so positioning/sizing is guesswork; (3) no visual debug feedback loop. Decision: a small dependency-free 3D-vector layer on the existing Math Box — near-plane clipping (shell), a camera + MVP pipeline (pure matrix builders in core/math3d, composition in shell), and a dev debug overlay. Determinism and the core/shell boundary preserved; zero new deps. SCENE-GEOMETRY FOLLOW-ON (see star-wars/docs/adr/0002-scene-geometry-surface-and-trench.md): after 11-1/11-2 the surface still renders as a triangle and the trench as a flat sliver — NOT a clipping/MVP bug (that plumbing is correct) but because the surface and trench were never built as a surface and a trench (a narrow Object_8 spike and a single flat floor tile). 11-4 adds a dev phase-jump so those scenes are reachable for verification (the gap that let the defect ship through two approved stories), 11-5 builds a procedural receding ground grid, 11-6 a walled receding trench channel, and 11-7 (optional) a Death Star body. Sequence 11-1 -> 11-2 -> 11-3 -> 11-4 -> 11-5/11-6 -> 11-7.

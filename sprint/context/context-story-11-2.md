## Title
Camera + MVP transform pipeline (model/view/projection + scale)

## Metadata

## Problem
Introduce a real model->view->projection chain. Add pure scaling() matrix and viewMatrix(camPos, orientation) builders to core/math3d.ts; build a per-entity model matrix = translation o rotation o scale; compose MVP = projection x view x model in the shell and transform vertices through it. Rewire render.ts to place models via {pos,rot,scale} through the camera, retiring the ad-hoc placement glue (SKIM_OFFSET, Z_SURFACE_PLACEMENT, surfacePlacement/trenchPlacement). The camera derives from sim state (the cockpit IS the camera); core stays pure. See star-wars/docs/adr/0001-3d-vector-render-pipeline.md (part B). Depends on 11-1.

## Acceptance Criteria

1. All red-baron HUD readout text — GUNS HOT, SCORE {n}, PLANE {n}, GAME OVER — renders through `@arcade/shared/font` (via `layoutText`/`charGlyph` stroked paths), NOT `ctx.fillText` with a `monospace`/system `ctx.font`. No `ctx.font = '...monospace'` assignment remains for HUD text in main.ts.
2. The pause/ESC overlay path (drawEscOverlay → @arcade/shared/esc-overlay) is UNCHANGED — this story only migrates the HUD readout.
3. Every HUD string is confirmed to lie within GLYPH_CHARS (uppercase/digits/space/`-,/_`); if any character is unsupported, it is surfaced, not silently dropped.
4. The migrated HUD preserves its screen-space placement and cabinet-green glowing-vector aesthetic (positions/anchoring equivalent to the current layout; glyphs stroked, consistent with the vector games' visual language).
5. `npm run build` and `npm test` stay green.

## Target API
`@arcade/shared/font` (v0.15.0, already pinned in red-baron/package.json):
- `layoutText(text, opts?)` → layout of `VecStroke` paths
- `charGlyph(ch): VecGlyph`, `hasGlyph(ch): boolean`
- `CELL_W = 16`, `CELL_H = 24`
- `GLYPH_CHARS = " 0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ-,/_"` — uppercase + digits + space + `-,/_` only.

## Test-Seam Trap (Critical for TEA)
Routing HUD text through the shared `@arcade/shared/font` subpath RELOCATES the observed text into the shared package. A `vi.mock` of a LOCAL font module — or a spy on `ctx.fillText` — will NOT observe it and the test will be vacuous.

**TEA MUST `vi.mock('@arcade/shared/font', ...)`** (the exact subpath main.ts imports) and assert on the CAPTURED args — the text strings and layout opts passed to `layoutText`/`charGlyph` — to prove the HUD routes through shared glyphs.

Because main.ts is the shell entry that mounts a real canvas, TEA should also confirm the test can drive main's HUD draw in isolation, or extract/observe the HUD-draw function; note any seam friction so Dev can expose a testable draw hook if needed.

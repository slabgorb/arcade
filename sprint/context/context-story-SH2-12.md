## Title
Extract @arcade/shared esc-overlay (pure pause gate + browser keybind card) from battlezone and adopt across all five canvas games

## Problem
Globalize battlezone's Escape-to-pause overlay (bz2-5) into one cabinet-wide behaviour, per the epic's verb/numbers rule: share the MECHANISM (Escape toggles pause -> the sim frame is frozen -> a dimmed panel + centered keybind card draws over the held world), keep the NUMBERS per-cabinet (each game's own keybind list, glow colour, dim opacity). Today only battlezone pauses. Its pure gate lives at battlezone/src/shell/pause.ts (INITIAL_PAUSED, isPauseKey, togglePaused, and stepUnlessPaused which freezes a frame by returning the SAME state reference so resume is deterministic); its browser overlay is drawPauseOverlay in battlezone/src/shell/render.ts (rgba black fill + drawScreenLines keybind card), wired by an Escape keydown EDGE (guarding e.repeat) in main.ts. Extract two pieces. (1) PURE @arcade/shared/pause: INITIAL_PAUSED, isPauseKey(key), togglePaused(paused), and a game-agnostic frozen-frame gate that takes the game's OWN step fn as an argument (it must import no game's sim), protected by the existing purity guard. (2) BROWSER @arcade/shared/esc-overlay: drawEscOverlay(ctx, w, h, opts) that dims the frozen scene and strokes a centered keybind card via @arcade/shared/font layoutText, with the card LINES, glow colour, and dim opacity passed in as per-cabinet parameters (no battlezone constants baked into the shared code). Then adopt in all five canvas games: battlezone re-points to the shared module behaviour-identical (its dual-tread card + green stay); tempest, asteroids, star-wars, and red-baron GAIN a pause, each wiring the Escape edge + frozen-frame gate + overlay and supplying its OWN keybind card + colour. Lobby is OUT OF SCOPE (a menu, not a game; cf. SH2-7). NOTE: red-baron is NOT yet on @arcade/shared/font (SH2-4/5/6 covered asteroids/star-wars/battlezone; tempest via SH2-2), so its keybind card must either ride a font migration or stroke via red-baron's local text path -- the implementing agent resolves and documents this. The overlay does not hard-depend on the SH2-8 glow primitive: it strokes card text via each game's existing shared-font path, and glowPolyline can replace the hand-stroke once SH2-8 lands.

## Problem
Found by Dev during tp1-25 (2026-07-14); the blocker behind that story's deviation D2. The ROM indexes invaders by X, a slot in the fixed INVAY array: INIINV (ALWELG.MAC:345-350) allocates NINVAD slots once and walks them with DEX, and a slot belongs to an invader for its whole life. The port has no such thing — s.enemies is a SPLICED array (sim.ts, `s.enemies.filter((_, i) => !deadEnemies.has(i))`), so an enemy's index SHIFTS whenever an EARLIER enemy dies. Any ROM rule keyed on the invader index therefore cannot be ported: MAYBLR's parity gate (2157-2160, which chases only on an ODD index — see tp1-28) is the one in front of us, and it is unlikely to be the last. Implementing it on array position would be WORSE than omitting it: a fuseball's chase would flicker on and off as UNRELATED enemies were shot, a behaviour the arcade has not got — it would trade a known, bounded deviation for an invented one. Give an enemy a slot id assigned at spawn and stable for its life, then implement MAYBLR's gate on it.

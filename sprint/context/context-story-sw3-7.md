## Problem
Every trench run is **byte-identical**: `spawnTrenchObstacles()` ignores the run
seed and returns one static hand-authored table. The cabinet instead builds each
run from a seeded PRNG "pie" so runs differ.

## Technical Approach (ROM contract — `sub_83A4`, recovered in RED)
The authentic trench is a **PRNG fixed-head + picked-tail** chain:
1. **Fixed head** — a skeleton copied verbatim from a ROM table (disasm `off_7C7E`
   → RAM `word_4B3F`; WSBASE.MAC `PIEXX` divider-panel format). Same every run;
   the "DIVIDER W/ CATWALK" panels put the **catwalk hazard in the fixed head**.
2. **Picked tail** — slots OVERWRITTEN with random picks from a ROM pool (disasm
   `off_7C9E`; WSBASE.MAC `TWDGXX`), indexed by a scaled PRNG byte
   `lda #$11; ldb PRNG; mul; asla` = `nextInt(rng, #entries)`.

The RAM pie is fixed-size (`RPIE..RPIEZ`) → chain LENGTH is seed-invariant, only
contents vary. Seed the tail from the `Rng` in `GameState` (mulberry32,
`@arcade/shared/rng`); `enterPhase(_, 'trench')` seeds from a LOCAL cursor
(`createRng(state.rng.seed)`) so the run RNG stays unmutated. No `Math.random`.

Sources: `reference/disasm/StarWars_annotated.lst` (ROM:83A4-83CD);
`~/Projects/star-wars-1983-source-text/WSBASE.MAC` (`NWBASE`/`GNBASE`).

## Scope
- In scope: seed the trench obstacle chain (fixed head + picked tail) from the run
  RNG so runs differ; keep length constant; keep the catwalk in the fixed head.
- Out of scope: the wave-gated pie SELECTION (pre-defined `TPIE` pies for early
  waves vs random `RPIE`) — logged as a non-blocking Delivery Finding.

## Acceptance Criteria (pinned by `tests/core/trench-variation.test.ts`, RED)
- Different seeds → **different** obstacle chains (regression: identical today).
- The chain has a **fixed head** (seed-invariant prefix, non-empty) and a **picked
  tail** (later slots vary by seed) — common prefix `0 < cp < length`.
- Same seed → **identical** chain (seeded, deterministic — never `Math.random`).
- Chain **length is seed-invariant** (ROM fixed-size RPIE).
- `enterPhase(_, 'trench')` does **not mutate** the caller's RNG (purity).
- Seed **0** is valid and still varies (guard the `|| default` falsy-seed pitfall).
- Every run keeps ≥1 catwalk; every obstacle is a known kind, downrange.

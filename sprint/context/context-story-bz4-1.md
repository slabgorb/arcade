## Problem
The ROM writes BOUNCE in three places; bz3-9 shipped only the obstacle-ram 0x3F. Add the getting-hit jolt: BOUNCE=0xFF on player death / windshield crack (BZONE.MAC:2337-2338) and on mutual player-enemy kill (BZONE.MAC:3363-3364), widening the bounce register from 0-63 to a full byte and reconciling its free-running behaviour across death. Also bob the horizon LINE with the mountains: bz3-9 bobbed only ridge/moon, but MOUNTS draws the horizon line from the same bounced beam origin (BZONE.MAC:1279-1282), so the ROM bobs them together. Surfaced by the bz3-9 review as an unfiled M-009 sibling plus an H-007 residual.

## Background

This story is the direct follow-up to **bz3-9 (COLLISION BOUNCE)**, which shipped:
- BOUNCE trigger: 0x3F (63) on obstacle collision (BZONE.MAC:2681-2682), rising edge via `motionBlocked && !motionBlockedLatch`
- BOUNCE decay: `>>1` per game frame in the 15.625 Hz `advanceRadar` loop (BZONE.MAC:2132-2134)
- Z-jolt effect: subtracts BOUNCE from every vertex depth (KLUDGE, BZONE.MAC:2110-2113), implemented via `tankView` eye translation
- Mountain bob: BOUNCE>>4 shifts the MOUNTS beam origin (BZONE.MAC:1265-1282), applied pre-projection elevation shift

**Key insight from bz3-9 review:** The ROM writes BOUNCE in THREE places, but bz3-9 only covered the obstacle collision (0x3F). This story adds the two death-path writes (0xFF).

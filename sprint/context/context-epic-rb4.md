Production code cites the doc as authority, so rb4-2 must retract it or it re-infects the next story. Note what is NOT wrong: the timebase is already correct (NMI 250 Hz / display 62.5 Hz / calc frame 96 ms = 10.4167 Hz, RBGRND.MAC:102 + CALCNT=0x18), and topology.ts is byte-exact — the picture ROM is genuinely .RADIX 10, so reading it as decimal was RIGHT. The transcriber applied decimal everywhere: correct for the geometry, wrong for every constant.

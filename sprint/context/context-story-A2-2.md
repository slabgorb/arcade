## Problem

This story is title-only with no formal description or acceptance criteria in the sprint YAML. The implementer must:

1. **Discover:** Locate the asteroids vector/stroke font implementation used for on-screen text (HUD score, GAME OVER, high-score entry, attract mode).

   - Find where font glyphs and advance-width are defined.

2. **Adjust:** Modify the inter-glyph letter spacing so text renders cleanly on screen.
   - Ensure readability without crowding or gaps.
   - Test across the full range of on-screen text (score display, messages, etc.).

## Technical Approach

### Font Strategy Gate (Decision Recorded)

A2-2 is formally gated by an unwritten ADR-0002 vector-font strategy decision (two incompatible approaches: Vector Battle TTF vs tempest's ROM stroke font). However:

- **Status:** User has explicitly approved proceeding.
- **Reason:** Sibling story bz2-2 ("arcade vector font") is already complete and has shipped a de-facto vector font implementation.
- **Scope of A2-2:** Does NOT implement a new font strategy. Instead, treats the established font from bz2-2 as settled and only adjusts its letter spacing.
- **This is a conscious bypass of the formal gate, not a blocker.**

### Implementation Path

1. Examine the asteroids codebase to find where the vector font is defined.
2. Identify the mechanism for inter-glyph spacing (character advance-width or kerning).
3. Adjust the spacing values to improve readability.
4. Test rendering of text in all UI contexts (HUD, game-over, attract mode, high-score entry).
5. Verify no visual regressions in other UI elements.

## Title
Finer control sensitivity — improve turret/aim granularity for precise targeting

## Problem
From the **bz2 playtest-followup epic** scope (verbatim from `epic-bz2.yaml`):

> "tank turret/aim control granularity is too coarse to aim precisely"

A live playtester found that turning/aiming the tank cannot resolve finely
enough to line a shot up on a distant enemy — each control input moves the
turret/heading by too large a step, so the crosshair skips past the target
rather than settling on it. This is a **control-feel / input-granularity** bug
in the Battlezone tank steering, not an enemy-AI or geometry problem (those were
addressed by bz2-9 and bz2-10, both done).

## Technical Approach
_Approach hints to be refined by TEA/Dev during RED._ Likely surface area:
Battlezone tank turn/aim input handling in `src/core` (deterministic steering
sim) and/or `src/shell` (input mapping). The dual-tread steering model already
exists — this story is about the granularity/sensitivity of the turn-rate
mapping, not adding a new control scheme. Keep it deterministic and testable in
`core`.

## Scope
- In scope: finer turret/aim granularity so precise targeting is possible —
  tuning the turn/aim rate mapping and/or adding a fine-aim capability.
- Out of scope: enemy AI, spawn geometry, camera/scale (bz2-3), pause/keybind
  overlay (bz2-5) — all separate stories.

## Acceptance Criteria
_TEA to formalize the ACs during the RED phase._ Anchor them to the epic intent:
a player can resolve aim finely enough to reliably line up a shot on a distant
enemy that previously could not be targeted, while coarse/fast traversal is not
degraded (fine control must not make normal turning sluggish).

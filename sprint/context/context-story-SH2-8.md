## Problem
First BROWSER subpath. Build the neon-glow primitive: withGlow(ctx, style, draw) sets strokeStyle/lineWidth/shadowColor/shadowBlur, runs the draw, then resets shadowBlur to 0 (the footgun every game hand-writes); glowPolyline(ctx, pts, style, close?) is the one-liner built on it. GlowStyle.stroke accepts string | CanvasGradient so richer looks are expressible. Deliberately no ctx.save/restore (mirrors the existing per-frame code). Migrate asteroids/star-wars/battlezone's single-pass polygon + text strokes onto glowPolyline/withGlow; per-cabinet blur/width constants stay in the games. Extend the purity guard so glow is a recognised browser subpath (exempt) while the pure core stays DOM-free.

## Technical Approach

### Key Context
- **First browser subpath:** This is the first @arcade/shared browser subpath. Existing subpaths (/math3d, /rng, /highscore, /loop, SH2-1/2/3 subpaths) are the model to follow.
- **Consumption model:** Each game pins @arcade/shared as a version-pinned git-URL TAG in node_modules (NOT a local working copy). Adopting the NEW /glow subpath requires: publish the new arcade-shared version → bump the git-URL tag in each game's package.json → reinstall. A fresh checkout may have @arcade/shared uninstalled.
- **ESM .js extension rule (CRITICAL):** Any browser subpath file that imports a sibling module MUST write the import with an explicit `.js` extension (e.g. `from './style.js'`). The tsc bundler-resolution compiles fine without it, but the shipped ESM artifact then fails native Node ESM (Vite hides the failure). This rule bit SH2-12; it will bite SH2-8 too. Rule #5 applies here.
- **Repos and roles:** arcade-shared is the library (produces @arcade/shared/glow); asteroids, star-wars, battlezone are the three "flat" games that migrate their single-pass polygon + text strokes onto glowPolyline/withGlow. Per-cabinet blur/width constants STAY in each game.
- **Purity guard extension:** The purity guard must be extended to recognize glow as a browser subpath (exempt from the DOM-free rule) while the pure core stays DOM-free.

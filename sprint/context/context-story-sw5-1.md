## Acceptance Criteria
- scripts/rom-models/wsobj.mjs parses `.WGD` routine bodies into the existing ConnectOp IR — PLOT n begins a run at point n (pen up), DRAWTO a,b,... draws a visible line to each point, BDRAWTO a,b,... is a blank move to `a` then delegates the rest to DRAWTO (same pen idiom as `.BD`/`.LD`), ENDPLOT ends the routine.
- Draw indices are parsed as DECIMAL — the macro forces it via the trailing-dot trick (`...1'.*10`). The `.RADIX 16` hex trap does NOT apply to these indices; do not apply it.
- `.WGD2` aliases the preceding `.WGD` routine BY REFERENCE, mirroring the existing `.WL2` behaviour (`.WGD TWR` / `.WGD2 GND` are one object).
- MOVD — and any other non-geometry state macro found inside a `.WGD` body — is recognised and skipped EXPLICITLY. It must never be handled by relaxing the throw-on-unknown-directive guard into silence; enumerate them.
- ORACLE — the two stroke sequences models.ts already quotes from the cabinet reproduce exactly, pinned as tests: the bunker's `BDRAWTO 14,15` and the tower's `BDRAWTO 7,9` / `BDRAWTO 7,8`. A human read these off this source by hand; the parser must agree.
- Every emitted edge index lands within [0, vertices.length-1] for all 24 objects. This is the guard on the anchor rebase — a uniform off-by-one would yield a plausible, wholly wrong wireframe.
- If a `.WGD` routine turns out to be genuinely unparseable (raw assembly, not macros), that object keeps hasDrawList false HONESTLY. Never guess an edge list.
- All 10 hasDrawList-false objects (GND, TWR, BNK, STB, WPN, WGA, WGB, WFF, WFG, PORT) flip to true after re-baking with `just bake-models star-wars`.
- The generated artifact's "edges not recoverable" caveat is removed or narrowed to whatever actually remains — leaving it stale makes it actively misleading.
- The contact sheet's punch-list regression pin covers 8 compared pairs (was 5), gaining PORT, STB and BNK.
- The red-baron oracle still passes (it guards the shared radix/number/pen core).

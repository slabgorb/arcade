## Problem
USER-REPORTED (2026-07-19, screenshot): every stamp is sideways — the SCORE/LEVEL glyphs and mushroom caps are turned 90 degrees clockwise. Root cause (Architect-verified in code): cp1-6's layout.ts maps CELL POSITIONS through ROT270 (col 0 rightmost, v=0 bottom) but buildAtlas() (src/shell/atlas.ts:69-81) bakes decodeStamp's grid in raw ROM-byte orientation — each tile's own pixels never get the rotation, so positions are right and pixels are sideways. The decode-orientation golden (tests/decode-orientation.test.ts, DIGIT_7 hand-derived in ROM byte order) pinned the bug as a regression fence, and the committed cp1-6 demo screenshot shows the defect (it must be re-taken). FIX DESIGN: rotation is SHELL policy — src/core/pictures.ts stays byte-exact against the vendored 136001.201/202 chips (cp1-3 AC, inviolable; do NOT rotate the core decode). Rotate each stamp into screen orientation at the atlas bake (the inverse of the observed clockwise turn); derive the exact mapping BY HAND from one asymmetric glyph BEFORE writing code; corroborate against MAME's gfx layout + ROT270 (schema-only claims, centiped.cpp). R3 test-hardening rides along (same test files): comment-strip the positive ?raw wiring scans, add a CSS-keyword/rgb-literal denylist to the no-invented-colour scan.

### ROM Quarry
The TIE object + wing-fragment models live in the Star Wars arcade **object-table** source, `WSOBJ.MAC`
(`.TITLE WSOBJ  OBJECT TABLES AND VECTOR DRAW ROUTINES`) — NOT `WSVROM.MAC` (that's the vector
picture ROM). The file was located and verified for this story; a clean copy is staged:
- **Staged source (use this):** `WSOBJ.txt` in this session's scratchpad —
  `/private/tmp/claude-501/-Users-slabgorb-Projects-a-1/bca258ca-a0e7-4d3a-9fc5-3b303dacf269/scratchpad/WSOBJ.txt`
  (LF-normalized; the original `WSOBJ.MAC` is CRLF and lives in the a-3 checkout's scratchpad).
- **⚠ Read gotcha:** the file contains embedded **NUL bytes**, so plain `grep` treats it as binary and
  silently returns nothing. Read it with the **Read tool** or **awk** (same class of gotcha as the
  red-baron CRLF ROM files), or force-text with `LC_ALL=C grep -a`.
- **Actual ROM labels ≠ the story-title names.** The story says `Obj_Tie_Wing_Frag_1/2/3` (a
  disassembly-style naming); in `WSOBJ.MAC` the objects are the terse Atari labels **`TIE`**,
  **`TIENT`**, **`WING`** (and fragment variants) under `.SBTTL OBJECT DRAW`. TEA maps title → ROM
  label during RED. Transcribe vertex/edge data **by ROM label, cited**, not invented.
- **Animation context:** Check how these fragments spawn/animate on TIE destruction (the TIE-destroyed
  animation this story wires into already exists in the star-wars sim/render).
- **Prior context:** See `sprint/archive/sw2-6-session.md` — it references this story and may contain
  pre-extracted wing-fragment quarry pointers from the prior TEA pass.

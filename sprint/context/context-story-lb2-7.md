## Problem
The front door is a bare centred column that says ARCADE; the design pass makes it a CABINET. Source of truth is the Claude design project 'Retro 80s lobby design' (claude.ai/design/p/f24e04ba-f4c1-40a4-8326-cebfca93c4e8, file 'Arcade Lobby.dc.html'). What it adds, all of it pure DOM + CSS on the existing tile-per-registry-entry structure: a receding cyan vector grid FLOOR with its mirrored CEILING (the Battlezone/Star-Wars horizon, plus its reflection through the half-silvered combiner — CSS perspective + repeating-linear-gradient, masked to fade at the horizon); four L-shaped cabinet BEZEL brackets framing the screen; a MARQUEE reading SLABGORB PRESENTS over a letter-spread VECTOR ARCADE wordmark over a blinking SELECT GAME; a per-tile MODEL SLOT (a bordered square carrying data-model-slot=<gameId>, which lb2-9 fills with a real vector model); the score line restyled to the design's HI-dot-SCORE typography; a screen-curvature vignette; a slow whole-page CRT flicker; and a footer with a blinking INSERT COIN plus the fan-tribute/not-affiliated disclaimer. The design's own prop defaults are authoritative for the CRT tweaks (curvature on, scanlines off). NOT in scope: the design's rotating top-five HIGH SCORES board (lb2-8 — it needs score data the cabinet cannot carry across origins yet) and real vector models in the slots (lb2-9). Keep the tiles real <a href> links so click, Tab+Enter, middle-click and open-in-new-tab all keep working for free, and keep the registry (src/core/registry.ts) the single source of truth for the games, their colours, their launch URLs and their control hints — the design hardcodes all four and must not be allowed to fork them.

## Acceptance Criteria
- The lobby renders the design's cabinet furniture — vector grid floor, mirrored ceiling, four corner bezel brackets, marquee, curvature vignette, CRT flicker and footer — and none of it intercepts clicks or focus (all decoration is pointer-events:none and out of the tab order).
- Every game listed on the page still comes from GAMES in src/core/registry.ts: the design's hardcoded titles, launch URLs, glow colours and control hints are NOT copied into the markup, and adding a game to the registry alone puts a correct tile on the page.
- Each tile keeps its real <a href> semantics — click, Tab+Enter, middle-click and open-in-new-tab all still launch the game, and the focus ring survives the restyle.
- Each tile carries an empty model slot marked data-model-slot="<gameId>", positioned and sized per the design, ready for lb2-9 to draw into without touching layout.
- The score line uses the design's typography but shows only data the lobby can actually prove: the real best score when readable (thousands-separated), NO SCORE when not. No fabricated name or score ships.
- Layout holds from a phone up to a wide desktop — the tile grid wraps, the marquee never overflows its line, and the floor/ceiling stay pinned to the viewport edges.
- npm run build (tsc --noEmit && vite build) and npm test both pass in lobby/.

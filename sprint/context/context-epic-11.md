# Epic 11 Context

## Title
star-wars 3D-vector render pipeline (clipping + camera/MVP + debug overlay)

## Overview
Fix the star-wars 3D vector renderer so geometry positions, sizes, and reads correctly — WITHOUT a 3D engine. Root causes (see star-wars/docs/adr/0001-3d-vector-render-pipeline.md): (1) no near-plane line clipping in wireframe.ts drawWireframe (edges straddling the camera plane are dropped, so the Death Star surface and trench render as triangles); (2) no camera/view transform or scale — placement is faked with ad-hoc world constants, so positioning/sizing is guesswork; (3) no visual debug feedback loop. Decision: a small dependency-free 3D-vector layer on the existing Math Box — near-plane clipping (shell), a camera + MVP pipeline (pure matrix builders in core/math3d, composition in shell), and a dev debug overlay. Determinism and the core/shell boundary preserved; zero new deps. Sequence 11-1 -> 11-2 -> 11-3.

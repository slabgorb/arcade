# Story jt2-1 — Process scheduler core: the tagged-union process list, frame naps, two classes, kill by id+mask, per-frame RNG stir

**Epic:** jt2 · **Points:** 2 · **Priority:** p1 · **Repo:** joust · **Workflow:** tdd

## Summary

Implement the cooperative scheduler core for Joust's deterministic simulation. GameState evolves from a frame counter into a process list that mirrors the ROM's 40-slot scheduler **semantics only** (no raw memory layout). Processes nap for whole frames, two scheduling classes (primaries then secondaries) step each frame, kill operations remove entire classes atomically, and the RNG advances once per frame independent of consumption — mirroring the authentic ROM's IRQ behavior.

## Technical Approach

**ROM Ground Truth:**
- Scheduler semantics: `SYSTEM.SRC:217/231/248` (PRISEC class dispatch)
- Process control registers: `RAMDEF.SRC:8-18` (PROCCR/SECCR)
- Process list structure: `RAMDEF.SRC:166` (PBLKM anchor)
- Process nap field: `RAMDEF.SRC:171` (PNAP, 1 nap = 1 frame)
- Kill-by-class operation: `SYSTEM.SRC:341-346` (KLPROC)
- RNG advance on timer: `SYSTEM.SRC:581-582` (IRQ-driven, not call-driven)

**Architecture:**
- `GameState` grows a `processes: Process[]` field — tagged-union to hold diverse process types (player, enemy, egg, wave machine, transporter)
- Each `Process` has: `id` (unique), `class` (primary vs secondary), `nap` (frame countdown), and type-specific fields
- Scheduler runs **two full passes per frame:** first all primaries (nap > 0 → decrement), then all secondaries
- `killByClass(targetClass)` removes all processes of a given class in one call
- RNG seeded by shell, advances once per frame via an integrate-every-frame divider (mimics the ROM's timer-driven cadence)

**Layout Rulings (BINDING):**
- Process layout anchors on `PBLKM` (`RAMDEF.SRC:166`) — never a searched ORG $0 (earlier instance exists; claim JT10-010)
- Pool arithmetic: 40 slots × 56 bytes/slot = 2240 bytes (NOT 2040)
- RULING: Semantics only — no raw 56-byte blocks, no PPC resume addresses, no memory image; cite layout facts from the dossier, never re-derive

**Migration Safety (jt1-5 Determinism Replay):**
- Player entities now join the process list, stepped from it
- Existing flight/ground stepping driven from the list without behavior change
- Seeded replay must reproduce bit-for-bit through the migration

**Purity Guard:**
- No `wall-clock` or `Math.random()` in core
- All RNG consumption is deterministic under seed

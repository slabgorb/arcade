## Problem
TIE fighters are currently rendered in red (#ff3b30) but should render in green to match the authentic 1983 Atari Star Wars cabinet color scheme. This is a cabinet fidelity bug affecting Wave 1 space combat visuals.

## Technical Approach
**File:** `src/shell/wireframe.ts`
**Constant:** `GLOW_FOR` object (lines 19-26)

The TIE fighter render color is defined in the shared `GLOW_FOR` record, which maps model names to glow colors used by both the game renderer and the model contact sheet. The current entry is:

```typescript
'TIE Fighter': '#ff3b30', // enemy red
```

To fix: change the hex color value from `'#ff3b30'` (red) to the authentic cabinet green (estimate: `'#00ff41'` or similar bright phosphor green). The `'Darth Vader TIE'` entry on line 21 should use the same green for consistency (both are TIE-class enemies).

**Impact:** The change affects only the render layer (shell). No core logic changes required.

## Problem
Found by the jt1-3 review (2026-07-20) and reproduced in this environment: strictPort does NOT protect a pinned port. With 127.0.0.1:5279 held, vite binds [::1]:5279 and starts normally — two dev servers, one port, NO collision error. This makes CLAUDE.md's pinned-port guarantee AND its own 'the port may belong to a different checkout' warning weaker than documented: you can serve and screenshot the WRONG CHECKOUT with no error at all, which is exactly the trap the docs warn about, minus the safety net. joust already carries the fix (host: '127.0.0.1' on server AND preview in vite.config.ts, joust#11 era) — port it verbatim to the seven siblings. Note the behavioural strictPort test in joust/tests/scaffold.test.ts occupies IPv4 only, which is why it was flaky before the host pin; siblings that copy the test should copy the fix with it. Also worth updating the orchestrator CLAUDE.md trap paragraph to say strictPort alone is insufficient.

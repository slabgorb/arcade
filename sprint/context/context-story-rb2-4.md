## Problem
Carryforward from rb2-1 (flight model). DISCHK proximity is a CALLER-supplied input in red-baron/src/core/flight.ts (FlightInput.proximity: near|mid|far, scale near 1.0 / mid 0.625 / far 0.375); rb2-1 hardcodes far in main.ts because there are no enemies yet. THIS story introduces the first enemy — wire the live nearest-enemy distance -> ProximityBand and feed it into FlightInput.proximity so the player control feel sharpens as the enemy closes (authentic DISCHK, findings §2). Ground-mode forced-slow band is rb3, not here.

Carryforward from rb2-3 (biplane render + LOD, src/core/biplane.ts — APPROVED, on develop). rb2-3 shipped the enemy biplane as PURE core building blocks but deliberately did NOT wire them into the runnable cockpit — that is THIS story's job. To render the spawned enemy: compose the model matrix translation(enemyWorldPos) then rotationZ(biplaneBank(enemy.turnRate)) (optionally rotationY for heading), form MVP = projection·view·model, select the model with biplaneLOD(depth) where depth = camera->enemy distance, call renderModel(model, mvp), and stroke the returned NDC SceneSegments as glowing vectors. Update the now-stale src/main.ts comment ("Still an EMPTY cockpit... no enemy/biplane geometry... blocked... a later rb2 story"). The weaving window-follower's ΔX toward P.OLIM/P.ILIM IS the turn-rate to feed biplaneBank, so the enemy banks into its weave with the same ±45° pfrotn coupling as the player horizon. GOTCHAS (rb2-3 reviewer findings): (1) build biplane models ONLY via biplaneLOD — renderModel trusts op.point to be in-bounds with no runtime guard, so a hand-built BiplaneModel with a mismatched points/connect pairing crashes in scene.ts toClip; (2) LOD_DISTANCE=1500 is an inferred tunable, not a ROM constant — retune against the real enemy world-scale during playtest; (3) renderModel/scene.ts projectSegment only cull edges with BOTH endpoints behind the eye — a biplane flown very close can flash a perspective-mirrored stray edge (near-plane clipping is unimplemented; candidate follow-up).

# Epic 7 Context

## Title
Arcade lobby v1

## Overview
The arcade's front door: a vector-style lobby on black that lists the games as glowing tiles, launches them, runs an attract-mode demo loop when idle, and shows per-game high scores. Matches the arcade visual language (glowing vector lines on black, Canvas 2D, no backend). Foundation for adding future games to the cabinet.

## Metadata
- **Epic ID:** 7
- **Repo:** lobby

## Architecture & Reuse
**Reuse the visual language, don't reinvent it.** The lobby is a fourth Canvas-2D
vector surface and should mirror the same glow/loop/storage patterns the games
already use — *patterns*, not a shared lib (no shared code yet, per the
orchestrator CLAUDE.md). The lobby owns navigation + launch; it does **not**
embed game logic.

The lobby scaffold (story 7-1) is done. Remaining work splits into:
- **Front-door critical path** (7-2 → 7-3 → 7-4): primitives, the tile grid built
  from the game registry, and selection/launch. This is what makes the arcade
  navigable.
- **Polish** (7-5 high scores, 7-6 attract mode) — P2.
- **Infrastructure** (7-7 canonical server) — P1, cross-repo (orchestrator).

## Cross-Story Constraints & Guardrails
1. **Lobby launches games; it does not contain them.** No game simulation in the
   lobby repo — only navigation, tiles, attract, and high-score display.
2. **Registry-driven.** Adding a game to the cabinet should mean adding it to
   `repos.yaml` — the lobby reads it, it doesn't hard-code a game list. Verify
   `star-wars` shows up once 7-3 lands.
3. **Serve from one place (7-7).** Until 7-7 lands, "it's shipped" is unverifiable.
   7-7 is the highest-leverage P1 in the epic and now must cover three games.
4. **Same visual language.** Glowing vectors on black, Canvas 2D, no backend.

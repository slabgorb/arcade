## Background

Cross-story constraints established by completed rb1 stories (cite the fidelity
spec `red-baron/docs/red-baron-1980-source-findings.md` for detail):

- **Frame cadence (rb1-2) — the load-bearing timing constraint.** Shipped Red Baron
  (`RBARON`+`RBGRND`, `FRMECNT=4` per the `RBARON.COM` link recipe) runs a 250 Hz NMI
  and a **62.5 Hz VG display refresh**, but the **sim advances one step per ~10.4 Hz
  calculation frame** (`CALCNT=0x18`=24 → 96 ms). **rb1-3+ must tick the flight sim at the
  calc-frame rate, not the display rate — ticking per display frame runs ~6× too fast**
  (the Red Baron analogue of the Asteroids ÷4 trap). Sub-dividers: enemy-shell launch ÷4,
  plane fire ÷2, off the `FRAME` counter. (R2BRON/R2GRND is a 50 Hz alternate revision — not shipped.)
- **Consume `@arcade/shared/math3d`; do NOT re-port the Math Box (rb1-1/rb1-2 ruling).**
  Feed it model vertex order Z,X,Y (`POINTP` = Z,2X,4Y); rotate Z-axis first, then Y (yaw)
  or X (pitch) by flight mode; divide-by-depth projection; `HORIZN=$40` horizon offset.
- **Flight-camera pipeline (rb1-2, seeds rb1-3):** `PLDELX` (turn-rate w/ inertia) pans the
  world + sets horizon bank `PFROTN=PLDELX×8` (clamped ≤`0x100`); `PLDELY` (11-step pitch)
  adjusts eye altitude `I4YPOS`, clamped `8*4..180*4`.
- **Open gap (rb1-2) — blocks authentic-plane render:** the picture-ROM *source*
  `RBPICS.MAC`/`RBCHAR.MAC` (the biplane face/line connect-lists + finished pictures) is
  **absent** from the quarry. The 42 vertices are in hand; the connect topology is not.
  A follow-up must recover those `.MAC` files or disassemble `037007.XXX`/`037006.XXX`
  before any authentic biplane geometry is rendered.
- **red-baron subrepo has NO GitHub remote** (local-only bootstrap policy): stories land via
  a **local merge to `develop`** — no push, no PR. Orchestrator: commit sprint tracking only.

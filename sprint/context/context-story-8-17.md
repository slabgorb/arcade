# Story 8-17 Context

## Title
Add cabinet HUD header — SCORE + score/lives, center SHIELD meter graphic + level, WAVE number — rendered in the vector font

## Metadata
- **Story ID:** 8-17
- **Type:** feature
- **Points:** 3
- **Priority:** p2
- **Workflow:** tdd
- **Repo:** star-wars
- **Epic:** Star Wars: vector cockpit shooter (Waves 0-5)

## Problem
The cabinet HUD is incomplete. Wave 1 gameplay (8-3/8-8/8-9) is now playable, but the player sees no persistent information about score, lives, shields, level, or wave number. The cabinet renders this information in a header row using the vector font, framed with glowing lines — an essential feedback loop and immersive element of the arcade experience. Today, the game shows no HUD at all (or a stub).

## Technical Approach

### Vector Font Foundation
The cabinet font is already available in the disassembly (`reference/disasm/Object_3D_Data.asm`, character geometry tables). The game has already ported the models in story 8-2. The font uses 3D vector strokes, matching the vector aesthetic.

### HUD Layout (Cabinet-Authentic)
The header row displays (left to right):
1. **SCORE label + current score value** (upper left) — e.g., "SCORE  1250"
2. **score/lives indicator** (left, below score) — e.g., "1/3" or "LIVES 2"
3. **SHIELD meter graphic + level** (center, large) — the shield bar drawn as a horizontal arc/meter (0–100%), with level number (1–10)
4. **WAVE number** (upper right) — e.g., "WAVE 1"

The header is framed by two horizontal glowing lines (top and bottom borders).

### Implementation Strategy

**Core (deterministic):**
- `src/core/hud.ts` (new) — exports text layout functions (no rendering) and string formatting helpers:
  - `formatScore(points: number): string` → "1250"
  - `formatLives(lives: number): string` → "3"
  - `formatWave(wave: number): string` → "1"
  - `formatLevel(level: number): string` → "1"–"10"
  - `formatShield(pct: number): number` — return the arc angle (0–360) or meter fill % for the shell to render
- Core state carries the necessary data already (`GameState.score`, `GameState.lives`, `GameState.shieldHealth`, `GameState.wave`, `GameState.level`). Verify these fields exist and are up-to-date; if not, add them.

**Shell (render/layout):**
- `src/shell/render.ts` — add `drawHudHeader(ctx, hud, game)` function:
  - Query the HUD strings/meter from `core/hud.ts`
  - Position text labels in screen space using the vector font (source: `core/models.ts` FONT_* entries)
  - Draw the shield meter as an arc (canvas `arc()` or stroked path) in the center
  - Draw the frame lines (two horizontal glowing vectors, top and bottom)
- `src/shell/loop.ts` — call `drawHudHeader` during the render phase (after game render, before compositing)

### Reference Sources
- **Cabinet disassembly:** `reference/disasm/Object_3D_Data.asm` — character/font geometry
- **Cabinet font layout:** study the original cabinet ROM behaviour (archive visual references) to position elements authentically
- **Tempest HUD:** `tempest/src/shell/render.ts` has a minimal score display; avoid cargo-culting; reuse only the vector-text pattern if it applies

### Acceptance Criteria
1. **HUD header renders on every frame** in Wave 1 space/surface/trench gameplay (and in attract/title if applicable), positioned at the top of the screen
2. **SCORE and lives display** — shows current score and remaining lives in the cabinet font, updated each frame from game state
3. **SHIELD meter and level** — a glowing horizontal arc or bar (0–100%) centered on the screen, with a numeric level (1–10) overlaid
4. **WAVE number** — displays the current wave (1–5) in the upper right
5. **Frame border** — top and bottom horizontal lines (glowing vector lines) frame the header row
6. **Authentic layout** — positions and styling match the cabinet visual (study reference disassembly or archive video)
7. **Responsive rendering** — the HUD scales/reflows correctly with the canvas viewport (no hardcoded pixel positions)
8. **npm run dev** — HUD visible and updating in live gameplay; all text legible
9. **npm run build** succeeds; **npm test** passes; no debug code
10. **Branch:** feat/8-17-hud-header in star-wars/develop

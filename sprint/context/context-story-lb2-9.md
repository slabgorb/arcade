## Title
Draw a real vector model in each lobby tile's model slot

## Problem
lb2-7 leaves every tile carrying an empty data-model-slot=<gameId> box; this fills it with the game's hero object drawn as a live glowing vector model — Tempest's blaster, a TIE fighter, an asteroid, a Battlezone tank, Red Baron's biplane — so the grid reads as an arcade row rather than a list of coloured rectangles. This is the single strongest thing the lobby can do to speak the cabinet's visual language, because it IS the cabinet's visual language.

DECIDED (SM, 2026-07-12): geometry is AUTHORED IN THE LOBBY, not extracted to shared. Hand-author tile-scale 2D vector silhouettes in the lobby (one per game) and draw them with @arcade/shared /glow (withGlow + glowPolyline). Rationale: ADR-0001 extracts on PROVEN duplication (its evidence is math3d.ts being byte-identical across two repos); tile art would be new geometry with exactly one consumer. The games' own model data is not reusable here anyway — it is 3D ROM geometry (star-wars models.ts is 743 lines, battlezone's 278), not tile-sized silhouettes. If a second consumer ever appears, extract then. Repos touched: lobby ONLY. No arcade-shared change, no release, no pin bump.

CORRECTION: the original 'lobby is still on @arcade/shared v0.4.0, so this story eats the version-bump cost' premise is STALE. lobby/package.json already pins v0.13.1, which exports /glow, /view, /font and /math3d. That cost was paid during lb2-2/lb2-3. There is no bump to sequence against.

star-wars' models contact sheet (/models.html) is still useful as a reference for eyeballing shape and scale. Static or slowly rotating is fine; this must not become a per-tile game loop that burns the visitor's battery on the front door.

## Acceptance Criteria
- Every tile's model slot draws that game's own hero object as a glowing vector model, in that game's registry glow colour. A game with no model degrades to the empty slot rather than a broken or blank canvas.
- The drawing goes through @arcade/shared/glow (withGlow + glowPolyline) rather than a re-implemented glow stack.
- The geometry is AUTHORED IN THE LOBBY as tile-scale 2D vector silhouettes (per the SM decision recorded in the description). arcade-shared is NOT modified: no new subpath, no release, no pin bump. lobby is already on @arcade/shared v0.13.1, which exports /glow.
- Animation (if any) is cheap and courteous — it pauses when the tab is hidden and does not run a full per-tile rAF game loop.
- The slot's presence or absence never shifts the tile layout lb2-7 established.
- npm run build and npm test both pass in lobby/.

Canonical arcade server: one authoritative way to serve the games (dev + production)

## Problem
The arcade orchestrator pattern has been cloned ~4 times under ~/Projects (a-2, a-1, arcade, plus a standalone tempest), and every copy can launch its own Vite dev server. Result: the 'production' server on port 5273 (arcade.slabgorb.com via the Cloudflare tunnel) was pinned to a standalone ~/Projects/tempest checkout that sat 5 commits behind, while active work happened in a different copy — so shipped work (6-2, 6-4) never appeared on the live server. There is no single source of truth for serving the games. Establish ONE canonical way to serve the arcade (lobby + game subrepos): a single authoritative checkout + launch command at the orchestrator, pinned ports, the tunnel wired to that checkout only, and the duplicate clones identified and marked non-authoritative or removed. Goal: starting work or viewing the live arcade never depends on guessing which of N copies is current. Known copies (2026-06-27): ~/Projects/a-2/tempest (current), ~/Projects/tempest (standalone, served 5273, stale), ~/Projects/arcade/tempest, ~/Projects/a-1/tempest.

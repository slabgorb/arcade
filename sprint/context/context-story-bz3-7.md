## Problem
Cluster C7. Subsumes D-001, D-002, D-003, D-005. D-001 depends on bz3-1 (frame rate). The defining arcade radar behaviour is missing: our core computes a sweep angle but never gates the blip list with it, so every target shows continuously instead of flashing as the rotating arm passes (D-003 — the ROM re-lights a blip only within a 0x0C = 16.875 deg window trailing the arm). No afterglow: the ROM sets BLIP=0xF0 then -8/frame = 30 frames = 1.92 s fade (D-002), which exceeds the sweep so blips pulse. The sweep is ~34% too slow: the ROM adds 0x0B (11) to SANGLE per frame over a 256-unit byte-angle circle -> 256/11 x 64 ms = 1489 ms/rev, where the clone uses 2000 ms (D-001; the 60 Hz false-match of 388 ms is ruled out). And no max range: the ROM drops the blip at TDIST >= 0x80, the clone clamps to the ring edge but never drops (D-005). DRADAR is JSRed once per frame at :510, so the sweep is FRAME-driven, not $INTCT-driven.

## Acceptance Criteria

- Sweep rate is corrected to +0x0B (11) byte-angle units per frame -> 1489 ms/rev (not the clone's 2000 ms), driven off the game frame.
- Blips past maximum range (TDIST >= 0x80) are dropped, not clamped to the ring edge.

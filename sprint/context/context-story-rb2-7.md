## Technical Approach

### Blocking Seam Gap — Enemy/Hit Kind Discriminant

rb2-6 landed the level-ramp infrastructure but left a critical gap: the `Hit{shell,target}` shape (guns.ts) and `Enemy` (enemy.ts) carry no lead/drone/blimp kind discriminant. rb2-6's lone-plane main.ts hardcodes kind='lead'. **rb2-7 MUST add a `kind` field to Enemy** (or wiring equivalent) so:
- `scoreKill(kind, depth)` picks the right score (drone flat 300, blimp flat 200, lead depth-scaled)
- The `PLNXCG` drone→lead promotion identifies which entity got promoted
- The `Hit` seam carries the kind through main.ts

This is not optional — the story's tests and wiring depend on it.

### Frame Cadence (Load-Bearing)
- Sim ticks one step per **calc-frame** (~10.42 Hz / 96 ms), **NOT** display frame
- src/core/timing.ts (`SIM_HZ`, `SIM_TIMESTEP_S`) encodes it
- Every wave/spawn/fire routine runs at calc-frame, not display refresh

### ROM Tokens to Implement

1. **MODECT plane-wave alternation:** alternate between lead and drone slots; object budget = 3 (1 lead + 2 drones max)
2. **MCOUNT inter-wave counts:** configurable delay (frame count) between waves; re-read before spawn check each calc-frame
3. **PLNLVL level-gated firing:** `PLNLVL[GMLEVL]` gates enemy fire percentage:
   - Level < 4: 0% (never fire)
   - Level 4: 50% (half cadence or 50/50 fire chance)
   - Level 5: 100% (always fire)
   - Fire rate halved by design (gated /2 from base cadence)
4. **Score-scaled spawn counts:** wave size ramps with score:
   - Score 0–299 (kills 0–1): 1 plane/wave
   - Score 300–999 (kills 1–2): 2 planes/wave
   - Score 1000+: 3 planes/wave
5. **Drone formation offsets:** drones spawn at fixed XY offsets from the lead's spawn point

## Problem
Found by TEA and Dev during tp1-25 (2026-07-14). docs/audit/findings/pair-1-alwelg-sim-enemies.json, finding W-023, states in its `claim`: 'MAYBLR only chases on an EVEN invader index (2157-2159).' It is FALSE. ALWELG.MAC:2157-2160 is `TXA / LSR / BCC LEFRIT / JSR FUCHPL`: LSR shifts bit 0 into the carry, BCC branches when the carry is CLEAR — an EVEN index — and that branch goes to LEFRIT, the COIN. The chase (FUCHPL) is what falls through, on an ODD index. The ROM's own comment on 2157 reads ';YES. ONLY IF INDEX IS EVEN' and is simply wrong — the auditor copied the comment instead of reading the branch. From there it propagated: audit claim -> tp1-25's story description -> tp1-25's AC-3, which asked for an 'even-invader-index gate'. This is the same disease as rb4-2 in red-baron: a doc that is cited as authority and is wrong. It cannot spread silently any more (tests/core/tp1-25.source-rules.test.ts pins the refutation with the call-site citation), but the doc still says it. Correct the claim. Do NOT touch the citations' line numbers or the remediated_by mechanics — only the prose that is false.

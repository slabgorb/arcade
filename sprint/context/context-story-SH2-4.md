## Title
asteroids — migrate HUD/framing text off the Vector Battle TTF onto @arcade/shared/font stroke-vectors

## Problem
Replace asteroids' loadVectorFont + ctx.fillText text rendering with layoutText(...) from @arcade/shared/font, stroking the returned glyphs with asteroids' existing local glow/stroke code (the shared glow primitive lands later in SH2-8). Delete public/fonts/*.ttf and the src/shell/font.ts TTF loader. Honour asteroids' letter-spacing (the A2-2 concern) via layoutText's opts. Behaviour-preserving except the intended visual change from TTF face to glowing stroke-vectors.

## Technical Approach
_Refined by TEA/Dev. Facts below are from the SH2-3 glyph-audit archive
(`sprint/archive/SH2-3-session.md`), the story that GATES this migration._

- **Asteroids' rendered text set = A–Z, 0–9, space, `_`** — all already covered by
  the shared VGMSGA table (SH2-3 added the `_ , /` gap glyphs). The `_` is the
  initials-entry echo `'_'.repeat(...)` at `asteroids/src/shell/render.ts:405`.
  `layoutText` covers **100%** of asteroids' rendered characters — no missing/
  placeholder glyphs. Asteroids has **no `▲`** concern (battlezone/SH2-6 only) and
  **no `©`/`(c)`** concern (copyright is DOM `<title>` only, never on canvas).
- **Shared module shape:** `@arcade/shared/font` (PURE, no DOM) exports
  `layoutText(text, opts?) -> {strokes, width}`, `charGlyph`/`hasGlyph`,
  `GLYPH_CHARS`, `CELL_W`/`CELL_H`. Asteroids' **letter-spacing (the A2-2 concern)**
  is expressed via `layoutText`'s `opts` — do not hand-roll spacing.
- **Consume, don't change, the shared font:** pin `@arcade/shared` at a git-URL ref
  whose SH2-3 font work is on `origin/develop`. If the git-dep lock is stale, force
  `npm install "@arcade/shared@github:...#ref"`. No edits to arcade-shared in this story.
- **Keep asteroids' local glow/stroke code** to stroke the returned glyphs — the
  shared glow primitive is a LATER story (SH2-8), out of scope here.
- **Non-blocking hardening note (do NOT fix here):** high-score `name` rows are
  validated only as `typeof name === 'string'`, so a tampered localStorage row
  could carry font-missing characters; in-code names are always uppercase A–Z.

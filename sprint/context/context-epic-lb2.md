# Epic lb2 Context

## Title
Lobby follow-ups — cross-origin high scores, plus front-end/design work

## Overview
Catch-all follow-up epic for the lobby, opened 2026-07-12 after the original lobby epic (epic 7) closed. First payload is the high-score break reported from production: the lobby tiles do not show each game's real best score and never update. Root cause is architectural, not a typo — lobby/src/shell/storage.ts reads localStorage on the LOBBY's own origin (arcade.slabgorb.com), while every game persists its table on its own origin (tempest.slabgorb.com, star-wars.slabgorb.com, ...). localStorage is origin-scoped, so in production the lobby reads a store no game ever writes; the shared @arcade/shared/highscore key/shape contract (SH-4) is correct but is being applied to the wrong origin. Separately, main.ts reads each score once while building the tile, so even a readable score is frozen for the life of the page. Fixing the first half requires a cross-origin decision under the standing no-backend constraint, so this epic gates implementation behind an ADR. Additional front-end/design stories will be appended to this epic from an in-flight Claude design pass.

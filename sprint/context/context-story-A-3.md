## Problem
The asteroids core (A-2) has a deterministic tick, seeded RNG, and entity
model, but the ship does not fly. This story implements the player ship's
flight model in the pure core: rotation, thrust, inertia, drag, and toroidal
screen-wrap, with constants tuned to the rev-4 ROM.

## Technical Approach
_Transcribed from the epic context (`sprint/context/context-epic-A.md`); TEA
refines exact constants during RED._

- Flight model lives in `src/core/` (state + sim), driven by the abstracted
  `Input` from A-2 (`left/right/thrust/...`). No DOM, no wall-clock, no
  `Math.random()` — all time enters as `dt`, all randomness via the seeded RNG.
- Wrap is a **sim** concern, not a render trick: positions are 2D world
  coordinates on a toroidal playfield (epic guardrail, shared with A-8).
- Constants are ROM-tuned per the epic authority chain: the rev-4 ROM
  disassembly at https://6502disassembly.com/va-asteroids/ (plus published
  analyses). **The `reference/` quarry does NOT exist in this checkout** — the
  quarry lives only in the checkout that created it (star-wars lesson), so
  source constants from the online disassembly and cite them in the code;
  tuned values must land in committed `core/` source.
- Copy patterns from star-wars (`star-wars/CLAUDE.md` is the blueprint); do
  not invent architecture or create shared libraries.

## Scope
- In scope: ship rotation, thrust acceleration along facing, inertia, drag
  decay, max-velocity cap, toroidal position wrap — in the pure core, with
  ROM-sourced constants and Vitest coverage.
- Out of scope: firing (A-4), rendering the ship (A-5), rock/collision wrap
  interactions beyond the ship (A-8), hyperspace (A-14), feel calibration
  against MAME (A-19).

## Acceptance Criteria
_Derived from the epic's fidelity bar; TEA formalizes as failing tests in RED._

1. Ship rotates left/right at the ROM rate; no rotation without input.
2. Thrust accelerates the ship along its facing; releasing thrust preserves
   velocity (inertia) subject to drag decay.
3. Velocity is capped at the ROM max; drag alone never reverses direction.
4. Ship position wraps toroidally on both axes in world space (sim-level).
5. Determinism: identical state + input + dt → identical result; tests use a
   fixed RNG seed and fixed dt.
6. Core purity holds: no DOM/canvas/wall-clock/`Math.random()` in `core/`.
7. Flight constants cite their disassembly source in the committed core code.
